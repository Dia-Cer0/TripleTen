HTML Document Structure

Now that you know some basic tags, it's time to talk about the structure of the entire webpage. The general structure of an HTML document looks like this:
HTML
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>Title of page</title>
</head>
<body>
  <h1>Main page heading</h1>
</body>
</html>
Let’s start at the top. First, the <!DOCTYPE html> tag that should be found at the top of all HTML documents. It exists to ensure that browsers follow the correct set of HTML specifications when they are rendering a document.
The <html> tag represents the root (or top-level element in an HTML document. All other elements must be children (also referred to as descendants) of the root element. The only elements allowed as immediate children of the root element are the <head> and <body> elements.
What’s the difference between the <head> and <body> elements? And why is the title inside the <head> element, while the <h1> is in the <body>? Here's an analogy to help you understand.
When you meet people on the street, all you can see is their external attributes, i.e. what they're wearing, the color of their eyes, their hairstyle, and so on. Still, we all know that there's more to someone than meets the eye. Everyone has their own thoughts and feelings, which are invisible to the rest of us. These are all hidden deep inside a person's head.
The same goes for a webpage, which has both visible and invisible parts. The invisible bit is represented by the <head> element. This element contains metadata about an HTML document: data that describes various aspects of the webpage. Information provided in the <head> of a document includes the page's title, links to external resources, and what language or character encoding to use. We'll describe this metadata and its uses more in future lessons.
The visible part is the <body>. What's inside the head of the webpage can affect the overall look of the body, much like with people.
